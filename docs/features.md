# 농사일지 (Farm Journal) — 기능 정의서

- 버전: v0.2 (개발 진행 중 — 실제 구현에 맞춰 갱신)
- 최종 수정: 2026-07-14

## 1. 목적

하루하루의 농사 일을 일기처럼 편하게 기록하고, 나중에 달력으로 훑어보거나 검색해서 찾아볼 수 있는 개인용 웹앱.
처음엔 혼자 사용하고, 나중에 원하면 다른 사람에게 공개할 수 있는 것을 염두에 둔다.

## 2. 설계 원칙

- **회사 업무 화면처럼 만들지 않는다.** 유형·작물·밭·수량·금액 같은 필드를 매번 강제로 채우게 하지 않는다.
- **일기 쓰듯 자연스럽게.** 작업 내용은 자유 텍스트로 줄줄이 쓴다.
- **파종·수확만 가볍게 태그.** 필요할 때 문장 끝에 탭 한 번으로 붙이는 정도로만 구분한다.
- **비용은 있을 때만.** 농기구 수리, 재료 구입처럼 돈이 나간 경우에만 별도로 입력한다.
- **농사와 무관한 것은 비고로.** 선거일, 서류 처리 등은 특이사항/비고 칸에 분리한다.
- **기록 없는 날은 달력에서 완전히 비워둔다.**

## 3. 데이터 모델 (단순화)

| 레벨 | 필드 | 비고 |
|---|---|---|
| 일지 (하루) | 날짜, 요일(자동), 날씨(선택), 특이사항/비고 | 비고는 농사와 무관한 메모용 |
| 작업 기록 (하루에 여러 줄) | 내용(자유 텍스트), 태그(파종/수확/없음) | 유형·작물·밭·수량 필드 없음 |
| 비용 기록 (선택, 독립 입력) | 내용, 금액 | 필요할 때만 별도 섹션에서 입력 |
| 사진 (선택, 하루에 여러 장) | 이미지 파일 | 일지 화면에서 바로 첨부 |
| 할일 (날짜 무관) | 내용(자유 텍스트), 완료 여부 | 일지와 별개의 단순 체크리스트 |

작물명·밭 이름은 별도 필드가 아니라 내용 텍스트 안에 자연스럽게 적고, 검색으로 찾는다.

## 4. MVP 기능 목록

### 4.1 오늘 기록 작성
- 날짜(기본 오늘), 날씨 칩 선택(맑음/흐림/비/눈, 선택 사항)
- 작업 내용을 줄 단위 자유 텍스트로 입력 (엔터로 다음 줄 추가)
- 각 줄에 "파종" 또는 "수확" 태그를 가볍게 붙이는 토글 (드롭다운 아님)
- "+ 비용 추가" 버튼 — 누를 때만 내용/금액 입력칸이 펼쳐짐
- "+ 특이사항/비고" — 맨 아래 별도 자유 텍스트 (선택)

### 4.2 달력 뷰 (주별 아젠다)
- 한 주(일~토)를 세로로 나열, 이전/다음 주 이동
- 기록 있는 날은 그날의 작업 기록·비용·비고를 말줄임 없이 전부 표시 (데스크톱·모바일 동일)
- 기록 없는 날은 날짜/요일만 표시 (완전히 비움)
- 날짜 클릭/탭 시 해당 날짜의 "오늘 기록 작성" 화면으로 이동 (조회와 수정을 겸함)

### 4.3 할일 목록 (사이드바)
- 달력 화면 옆(데스크톱은 왼쪽, 모바일은 위쪽)에 위치하는 별도의 간단한 체크리스트
- 특정 날짜와 무관하게 등록 — 할 일 내용만 자유 텍스트로 입력
- 체크박스로 완료 표시, 삭제 가능

### 4.4 목록 / 검색
- 전체 텍스트 키워드 검색
- 필터: 전체 / 파종 / 수확 / 비용 (4개뿐, 작물·밭 필터 없음)
- 기간별 필터(이번 달/지난 달/직접 지정)

## 5. 확장 기능 (후순위, MVP 이후)

- 작물/밭 마스터 관리, 파종일 기준 "OO일차" 자동 계산
- 날씨 자동 연동 (위치·날짜 기준 API)
- 통계/리포트 (작업 빈도, 지출 추이 그래프)
- 오프라인 지원 (PWA)
- 리마인더/알림
- 데이터 내보내기 (엑셀/PDF)

## 6. 공개 전환 단계 (추후)

- 로그인/권한 분리 (소유자만 편집, 방문자는 열람만)
- 게시글 단위 공개/비공개 토글
- 댓글 또는 소통 기능
- SEO, 공유 링크
